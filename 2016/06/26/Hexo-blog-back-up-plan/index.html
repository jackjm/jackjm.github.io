title: Hexo blog back up plan
date: 2016-06-26 22:02:00
tags:

Hexo is an efficient static pages generator.

Here’s a backup plan for Hexo system:

  1. Create a symbolic link from the Hexo source/_posts directory to a Dropbox directory.
    So all the source files, i.e. markdown files, of the Hexo will be synced to Dropbox server, and
    all editing versions is preserved by the “previous versions” function of Dropbox.

  2. Using the Hexo backup plugin Git-backup.
    Install the git-backup plugin, https://github.com/coneycode/hexo-git-backup, and manage the backup
    with init.d or supervisor.

  3. Backup with Crashplan
    Backup the whole Hexo directory to another computer/server with crashplan.

  4. Deploy the blog to multiple host servers.
    There are many Pages hosts:
    Github Pages
    Gitlab Pages
    http://docs.gitlab.com/ee/pages/README.html
    Bitbucket Pages
    Oschina Pages
    https://www.oschina.net/news/73980/gitosc-pages
    Coding.net Pages
    https://coding.net/help/doc/pages/

http://sebduggan.com/blog/deploy-your-website-changes-using-git/
